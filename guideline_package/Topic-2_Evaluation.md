## 1. Evaluation Criteria

> **Note:** These criteria should help students determine whether they solved the problem correctly and whether the solution is of good quality. It should be applicable (ideally) to any problem in your topic.

### General Evaluation Criteria

> **Note:** These are generic examples only (i.e., for your guidance)

- **Correctness:** Does the solution produce the expected output for all test cases?
- **Completeness:** Does the solution address all requirements specified in the task description?
- **Code Quality:** Is the code well-structured, readable, and maintainable?
- **Efficiency:** Does the solution use appropriate algorithms and data structures?
- **Guideline Adherence:** Does the solution follow the guidelines from `Topic-XX_Guidelines.md`?
- **Edge Cases:** Does the solution handle edge cases and error conditions appropriately?
